# 🦆 Охота на уточек — WebAR-квест по кампусу School 21

Веб-приложение дополненной реальности. Гость сканирует QR → открывается сайт → ходит по кампусу, наводит камеру на стикеры → на них появляются 3D-уточки → ловит их тапом. Каждая уточка ловится один раз, ведётся счёт. Работает **в браузере** на iOS и Android, ничего не нужно устанавливать.

Технологии: [MindAR](https://hiukim.github.io/mind-ar-js-doc/) (image tracking на TensorFlow.js) + [A-Frame](https://aframe.io/). Хостинг — статический (GitHub Pages), бэкенд не обязателен.

## Как это работает

```
QR на входе ──▶ сайт (GitHub Pages)
                  │
                  ▼
            экран старта (имя, «Начать охоту»)  ← кнопка = жест для запуска камеры на iOS
                  │
                  ▼
            камера + MindAR ищет стикеры
                  │  нашёл стикер #i
                  ▼
            показывает 3D-уточку  ──tap──▶  поймал (учёт в localStorage, без дублей)
                  │
                  ▼
            счётчик растёт, в конце — экран результата для организатора
```

Структура проекта:
```
duck-hunt-ar/
├── index.html              # разметка + AR-сцена
├── css/style.css           # тёмный «терминальный» UI
├── js/app.js               # вся логика (правь CONFIG вверху файла)
└── assets/
    ├── duck.glb            # 3D-уточка (можно заменить)
    └── targets.mind        # ← ВЫ создаёте его из своих стикеров (шаг 2)
```

## Быстрый старт за 6 шагов

1. Сделать стикеры-картинки → [как сделать хорошие](#шаг-1-стикеры--как-сделать-хорошие-это-главное-для-качества).
2. Скомпилировать их в один `assets/targets.mind` → [компиляция](#шаг-2-скомпилировать-targetsmind).
3. В `js/app.js` выставить `numTargets` = число стикеров → [настройка](#шаг-3-настроить-число-уточек).
4. Залить папку на GitHub Pages → [деплой](#шаг-4-деплой-на-github-pages).
5. Сгенерировать QR на адрес страницы → [QR](#шаг-5-qr-код-на-вход).
6. Распечатать стикеры + QR, развесить по кампусу. Готово.

> Камера в браузере работает **только по HTTPS**. GitHub Pages даёт HTTPS из коробки — поэтому он идеально подходит. Локально для теста используйте `https://` туннель (ngrok/cloudflared) или `localhost`.

## Шаг 1. Стикеры — как сделать хорошие (это главное для качества!)

MindAR узнаёт картинку по «фичам» — уникальным контрастным точкам. **От качества стикеров зависит 80% успеха.** Плохая картинка не отследится даже на мощном телефоне.

**Хорошая картинка-цель:**
- насыщенная деталями, с высоким контрастом (фотографии, иллюстрации, узоры);
- **асимметричная** — без повторяющихся паттернов и зеркальной симметрии;
- не однотонная, без больших пустых зон, без крупного градиента;
- близкая к квадрату или 4:3 (очень вытянутые хуже).

**Плохо:** логотип на белом фоне, текст на пустом фоне, минималистичная иконка, повторяющийся орнамент, очень тёмная картинка.

**Печать и размещение:**
- печатайте **матово** (глянец бликует → трекинг рвётся);
- крупно: ориентир — сторона **15–25 см**, чтобы ловилось с 0.5–1.5 м;
- вешайте на ровную поверхность, на уровне глаз, в **хорошо освещённых** местах без прямого блика;
- идея: сделайте каждый стикер своим (разные уточки/номера) — игроку нагляднее, и фичи разнообразнее.

Проверить качество картинки можно прямо в компиляторе (шаг 2): он показывает число фич-точек. Чем больше красных точек — тем стабильнее трекинг.

## Шаг 2. Скомпилировать `targets.mind`

Все картинки-стикеры собираются в **один** файл `targets.mind`.

1. Открыть онлайн-компилятор: **https://hiukim.github.io/mind-ar-js-doc/tools/compile/**
2. Загрузить **все** картинки стикеров **сразу** (порядок загрузки = индексы 0,1,2,…).
3. Нажать **Start**, дождаться, нажать **Download** — получите `targets.mind`.
4. Положить файл в `assets/targets.mind` (заменив заглушку).

> ⚠️ **Порядок важен.** Картинка, загруженная первой, станет уткой `#1` (индекс 0), вторая — `#2` и т.д. Это же число картинок впишите в `numTargets` (шаг 3).

Файл `targets.mind` целиком оффлайновый — после загрузки игра не зависит от компилятора.

---

## Шаг 3. Настроить число уточек

Открыть `js/app.js`, вверху в `CONFIG`:

```js
const CONFIG = {
  numTargets: 8,   // ← поставьте число картинок в targets.mind
  ...
};
```

Если у вас, например, 12 стикеров — поставьте `numTargets: 12`. Если поставить больше, чем есть картинок, лишние просто никогда не появятся (не страшно). Если меньше — часть уточек будет недоступна.

## Шаг 4. Деплой на GitHub Pages

**Через сайт GitHub (без терминала):**
1. Создайте новый репозиторий, например `duck-hunt-ar` (public).
2. Загрузите в него **содержимое** этой папки (файлы `index.html`, папки `css`, `js`, `assets`) — кнопка **Add file → Upload files**.
3. **Settings → Pages → Build and deployment → Source: Deploy from a branch**, ветка `main`, папка `/ (root)` → **Save**.
4. Через 1–2 минуты появится адрес вида `https://ВАШ-НИК.github.io/duck-hunt-ar/`.

**Через терминал:**
```bash
cd duck-hunt-ar
git init && git add . && git commit -m "duck hunt ar"
git branch -M main
git remote add origin https://github.com/ВАШ-НИК/duck-hunt-ar.git
git push -u origin main
# затем включите Pages в Settings → Pages (как выше)
```

> Если страница открывается, но уточек нет и в консоли ошибка про `targets.mind` — вы не залили файл целей (шаг 2) или указали неверный путь.

## Шаг 5. QR-код на вход

Сгенерируйте QR на адрес страницы (`https://ВАШ-НИК.github.io/duck-hunt-ar/`) любым генератором (например, qr-code-generator или `qrcode` в Python). Распечатайте крупно у входа.

> 📱 На плакате с QR добавьте подпись: **«Откроется в браузере. Если открылось в приложении — нажмите ⋯ → Открыть в Safari/Chrome»** — встроенные браузеры мессенджеров часто не дают камеру (приложение это определяет и подскажет, но лучше предупредить заранее).

## Настройки (CONFIG)

Все настройки — вверху `js/app.js`:

| Параметр | Зачем |
|---|---|
| `numTargets` | число стикеров (= число картинок в `targets.mind`) |
| `duckScale` | размер уточки на стикере. Меньше — мельче. По умолчанию `0.005` |
| `duckOffsetY` | подъём уточки над центром стикера |
| `duckOffsetZ` | насколько уточка «выезжает» из стены к зрителю |
| `bobAmplitude` | амплитуда покачивания вверх-вниз |
| `spin` | `true` — уточка медленно крутится (удобно: не зависит от того, куда смотрит клюв) |
| `leaderboard.*` | таблица лидеров — см. отдельный раздел |

**Подгонка размера/позиции уточки** делается «на глаз» прямо на устройстве: открыли на телефоне, навели на стикер, не нравится размер — поменяли `duckScale`, обновили. Это нормально, что нужно пару итераций — зависит от пропорций ваших стикеров.

## Производительность на слабых телефонах

Уже сделано в проекте:
- `maxTrack: 1` — трекается только одна картинка за раз (**главная** экономия CPU/GPU);
- лёгкая модель уточки (~120 КБ, low-poly, одна текстура);
- рендер без сглаживания, `precision: mediump`, `powerPreference: low-power`;
- ловля по тапу в любую точку экрана, а не рейкастом по 3D — меньше вычислений и **невозможно промахнуться**;
- свой лёгкий UI вместо тяжёлых встроенных оверлеев.

Если всё равно тормозит на совсем старых устройствах:
- уменьшите/упростите модель уточки (см. ниже) или текстуру;
- развесьте стикеры в хорошо освещённых местах (плохой свет → MindAR дольше «думает»);
- крайний случай для максимальной живучести — перейти на маркеры AR.js или на QR-триггер (см. сообщение в чате, там разобраны оба пути).

## Таблица лидеров (опционально, Firebase)

По умолчанию **выключена** — игра считает уточек локально, и в конце победитель просто показывает экран организатору. Этого достаточно для маленького ивента и не требует бэкенда.

Если нужна **общая** таблица лидеров на всех (по-прежнему без своего сервера — пишем напрямую в Firebase из браузера):

1. Создайте проект на https://console.firebase.google.com → добавьте **Realtime Database** (test mode для начала).
2. **Project settings → Your apps → Web** → скопируйте `firebaseConfig`.
3. В `js/app.js` в `CONFIG.leaderboard`:
   ```js
   leaderboard: {
     enabled: true,                       // ← включаем
     firebaseVersion: "10.12.2",
     firebaseConfig: { /* вставьте свой объект */ },
     eventId: "school21-ducks",
   }
   ```
4. Готово: после каждой поимки счёт улетает в базу, а кнопка **«Таблица лидеров»** на экране «Готово» показывает топ-20.

> ⚖️ **Честно про античит.** Вы делаете ивент в школе программистов — счёт на клиенте (и localStorage, и прямую запись в Firebase) технически подделать несложно. Варианты по возрастанию серьёзности: (1) положиться на дух мероприятия и проверять физически топ-3; (2) обфусцировать; (3) если нужна настоящая защита — добавить серверную валидацию (каждая поимка подтверждается одноразовым кодом со стикера). Для квеста на час обычно хватает варианта (1).

## iOS / Android: что важно знать

- **HTTPS обязателен** — иначе камера не запустится. GitHub Pages = HTTPS. ✔
- **iOS**: камера стартует только после жеста пользователя — поэтому камера включается по кнопке «Начать охоту», а не сама. ✔ Работает в Safari (и Chrome на iOS, который под капотом Safari).
- **Встроенные браузеры** (Instagram, Telegram, VK, TikTok и т.п.) часто **блокируют камеру**. Приложение это определяет и просит открыть в Safari/Chrome, но предупредите гостей на плакате. ✔
- **Android**: работает в Chrome. В очень старых WebView возможны просадки — см. раздел про производительность.
- Разрешение на камеру запрашивается каждый раз заново, если пользователь его не «запомнил» — это нормально.

## Траблшутинг

| Симптом | Причина / решение |
|---|---|
| «Нет файла целей» | Не залит `assets/targets.mind` или неверный путь. Скомпилируйте (шаг 2) и положите в `assets/`. |
| Камера не включается | Нет HTTPS, или доступ к камере запрещён, или открыто во встроенном браузере. Откройте в Safari/Chrome, разрешите камеру. |
| Стикер не распознаётся | Слабая картинка-цель (мало фич), блик от глянца, плохой свет, далеко/под углом. Матовая печать, крупнее, ближе, лучше свет, более «фактурная» картинка. |
| Уточка чёрная | Не загрузился свет или модель. Проверьте, что `duck.glb` на месте; в сцене есть `a-light`. |
| Уточка гигантская / крошечная | Поменяйте `duckScale` в CONFIG. |
| Уточка «утоплена» в стену или висит мимо | Поменяйте `duckOffsetZ` / `duckOffsetY`. |
| Поймал, а счётчик не изменился | Уточка уже была поймана (каждая ловится один раз). Сбросить прогресс: очистить данные сайта в браузере. |
| Хочу сбросить прогресс игрока | DevTools → Application → Local Storage → удалить ключи `duckhunt:*`. |

## Замена модели уточки

Подойдёт любой `.glb` (желательно low-poly, < ~300 КБ, с одной текстурой). Положите как `assets/duck.glb`. Бесплатные модели — Poly Pizza, Sketchfab (фильтр Downloadable + CC0), Quaternius. После замены подгоните `duckScale` (у разных моделей разный исходный размер).

Текущая модель — классическая «резиновая уточка» Khronos: идеальна тематически (rubber duck debugging) и очень лёгкая.
